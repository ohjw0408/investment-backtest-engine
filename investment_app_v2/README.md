지금 현재 내 앱에서는 티커를 입력하면 입력한 티커의 개수만큼 슬라이더가 생기고, 그 슬라이더의 비중을 조절하여 n년부터 n년까지 그 포트폴리오의 성과와 n년 롤링 수익률을 확인할 수 있어 (역시 슬라이더로 설정 가능). 월간/일간으로 데이터 정밀도를 체크할 수 있는 박스와 월간/연간으로 리밸런싱 주기를 선택하는 것도 가능해. 일단 이 기능이 있어. 앞으로 내가 목표하는 건 1. 원하는 자산을 검색창에서 검색하여 선택할 수 있고, 증권사 앱처럼 그 자산(주식아니 코인,etf등)의 정보(주가차트,거래량,배당,실적,재무제표 등)를 확인이 가능한 것. 2. 여러 자산을 선택해 포트폴리오를 짜 나만의 포트폴리오를 저장할 수 있는 것, 그렇게 짠 포트폴리오의 n년부터 n년까지의 성과, 최대낙폭, 대표지수와의 성과 비교, n년 롤링 데이터, 개별자산 기여도, 회복까지 걸린 시간 등의 데이터를 그래프와 도표로 직관적으로 보여줌과 동시에, 비중을 실시간으로 조절하며 데이터가 어떻게 변하는지 사용자에게 보여주는 것. 그리고 n년 후에 지금 얼마를 투자했을 떄 얼마의 자산 증식과 얼마의 배당금을 기대할 수 있는지 보여주는 것 3. 그렇게 개인이 만든 포트폴리오로 투자했을 때, 매일(혹은 매달 특정일 혹은 매년 특정일 혹은 특정 조건이 만족되엇을 때) 리밸런싱 알람을 보내주어 무슨 종목을 어떤 증권사의 어느 걔좌에서 얼마나 사고 팔면 되는지 알림을 보내주는 것 4. 그렇게 개인이 투자를 지속했을 떄 성과가 얼마나 되는지 사용자에게 보여주는 것.최종적으로는 증권사의 일부에 통합되어 특정 증권사 앱의 새로운 기능이나 상품이 되는 것이야. 일단 구현 순서는 2번,1번 3번 4번으로 하고 싶어.3번,4번은 마이데이터 연동같은 1인개발로는 힘든 제약이 있을 것 같아서. 이 정도의 확장을 의식하고 시작하는 초대형 프로젝트라 명확한 블록화와 안정성이 중요할 것 같아

📝 Project: Quant-Master Pro (3-Year Roadmap)
1. 서비스 목표
1단계: 고성능 데이터 로딩 및 로컬 캐싱 엔진 구축 (1950년~현재 데이터 지원)

2단계: 전 세계 주식/ETF 종목 정보 통합 검색 및 필터링 시스템 구현

3단계: 사용자 맞춤형 포트폴리오 시뮬레이터 및 성과 분석 리포트 생성

4단계: 리밸런싱 알림 및 포트폴리오 관리 자동화 준비

2. 시스템 아키텍처 (Layered Architecture)
미래의 확장성을 위해 기능을 3개의 핵심 모듈로 분리하여 설계합니다.

Data Layer (modules/data_engine.py)

로컬 캐시(CSV/DB)와 원격 서버(FDR) 간의 데이터 흐름 관리.

1970년 이전 데이터 처리 시 시스템 예외 상황(Overflow) 자동 우회.

상장 전 빈 구간을 지수(Benchmark)로 메꾸는 스마트 백필링 로직.

Analysis Layer (modules/analyzer.py)

환율 변동을 반영한 원화 환산 수익률 산출.

리밸런싱 주기(월간/연간)에 따른 포트폴리오 가치 추적.

CAGR, MDD, 샤프 지수 등 퀀트 성과 지표 계산.

Info & UI Layer (app.py, modules/info_engine.py)

종목명/티커 검색 및 상세 정보(섹터, 배당 등) 전시.

Streamlit 기반의 반응형 대시보드 및 로그 스케일 시각화.
