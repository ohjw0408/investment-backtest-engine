# 수익 모델

## 우선순위 순서

### 1순위 (Best): 자산운용사 B2B2C 파트너십

**개요**: 시뮬레이션 결과 화면에서 운용사의 실제 ETF/펀드/TDF 상품을 자연스럽게 노출.

- 예시: 은퇴 계산기 결과 → "이 포트폴리오와 가장 유사한 미래에셋 TDF2045 상품"
- 예시: 백테스트 결과 → "이 자산배분과 유사한 삼성 올웨더 ETF"
- 타깃 파트너: 삼성자산운용, 미래에셋, KB자산운용, 한투운용
- 수익 구조: 광고비 + CPA (계좌 개설/상품 가입 수수료 쉐어)

**왜 이게 Best?**
- 사용자 니즈(시뮬 결과)와 광고(관련 상품)가 완벽 매칭 → 클릭률 높음
- 운용사도 신규 계좌 유치에 혈안 → 협업 동기 충분
- 앱이 커질수록 협상력 높아짐 (트래픽이 레버리지)

**실행 조건**: 먼저 유저가 있어야 운용사가 관심 가짐. 유저 수 최소 1만 이후 접근.

---

### 2순위 (Alternative A): 저가형 구독 모델

**구조**: 월 1,000~1,800원 구독

- 무료 플랜: 모든 계산 기능 사용 가능, 광고 표시
- 유료 플랜 혜택:
  - 광고 제거
  - 리밸런싱 알림 기능 (비중 이탈 시 알림)
  - 계산 결과 저장 + 히스토리
  - (추후) 증권사 API 자동 리밸런싱

**왜 1,000~1,800원?**
- 커피 한 잔보다 저렴 → 결제 마찰 최소화
- 구독자 1만명 × 월 1,500원 = 월 1,500만원 ARR

---

### 2순위 (Alternative B): Google AdMob 광고

**구조**: 시뮬레이션 계산 대기 시간(5~10초)에 광고 노출

- 앱 특성상 계산 대기가 자연스럽게 발생
- 광고 강요 없이 자연스러운 노출
- CPM 기준 트래픽 비례 수익

**제한**: 단독으로는 수익 작음. 구독 전환 동기 부여 용도로도 활용.

---

### 장기 (Long-term): 증권사 API 연동 자동 리밸런싱

**개요**: 사용자가 앱에서 설정한 포트폴리오대로 증권사 계좌에서 자동으로 리밸런싱.

- 파트너: 키움증권, 삼성증권, 토스증권 등 API 개방 증권사
- 수익: 거래 수수료 쉐어 + 월 구독료 프리미엄 플랜
- 이게 완성되면 "완전 패시브 투자 플랫폼" → 카테고리 리더

**실행 조건**: 증권사 API 계약 필요. 규모가 있어야 협상 가능. 장기 목표.

---

## 수익화 로드맵

| 시기 | 액션 |
|---|---|
| 베타 (지금~2026.10) | 무료 배포, 유저 확보, 피드백 수집 |
| 2026.11 런칭 | AdMob 광고 + 구독 모델 동시 런칭 |
| 2027 상반기 | 유저 수 기반으로 운용사 B2B 제안 시작 |
| 2027 하반기+ | 운용사 파트너십 or 증권사 API 연동 |

---

## 현재 고민 (미결정)

- 구독 가격대 확정 필요 (1,000 vs 1,500 vs 1,800원)
- 무료 플랜에서 기능 제한 범위 설정 (너무 많이 제한하면 유저 이탈, 너무 적으면 구독 전환 안 됨)
- 광고가 UX를 해치는 수준이 되지 않도록 균형 필요
