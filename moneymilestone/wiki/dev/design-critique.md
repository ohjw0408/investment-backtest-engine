방법 = 페이지당 6컷(데스크탑 1280/1920·모바일웹 390 × 라이트/다크) 육안 크리틱 → 오너 승인 → 구현.
앱 WebView는 390 근사(최종 실기기). 기준선 = 코인베이스 아키타입 + 토스급 밀도. 리스킨 금지.

## 배치 1: 홈 (2026-07-03) — 오너 리뷰 반영 완료

**오너 판정**: D1·D2 = 철회(테스트 계정 산물 — D1 금액마스킹+원 조합은 오너 실계정에 없음, D2 "구멍"은 다음 슬라이드 존재 때문). D3 = 기각(현행 유지). **D4·D5 승인 → 구현 완료.**
**오너 신규 2건**: ①위젯 스와이프 모멘텀 과다(여러 장 넘어감) → `scroll-snap-stop: always` ②코스피 밤 "데이터 없음" → BUG-KOSPI-NIGHT-NULL로 별도 수정(bugs.md).
**D2 재승인·개편 완료(오너 후속 지시)**: 오너 재검토 — 빈 칸 + 홀수 항목 페이지의 선 엉킴 + 긴 이름 시 칸 폭주/잘림 = 셀·보더 구조 자체 문제. 선택지 3안 중 **②모바일=리스트 행 / 데스크탑=타일 그리드** 채택.
- 데스크탑 `_renderGrid`: 보더 그리드 → gap 타일(둥근 카드·min-height 118·이름 nowrap+ellipsis+title). 빈 슬롯 = 투명 gap.
- 모바일 캐러셀: 2열 그리드+인덱스 계산 인라인 보더 → **리스트 행**(이름/코드 좌 ellipsis·소형 스파크 64px·가격/등락 우측 정렬). 행 밑줄 1종 — 선 엉킴·빈 칸 개념 소멸. PF: 코드는 "내 포트폴리오" 표기.
- D4 변수 핵 제거(보더 로직 소멸로 불필요) → 타일/행 다크 보더 #3a3f49 직접 지정.
- 검증: 타일 5/행 5·ellipsis 적용·라이트/다크 4컷·jsErr 0.

**방법론 교훈**: 크리틱은 오너 실계정 데이터 기준으로 봐야 — 테스트 계정 산물(마스킹·커스텀 위젯명·시드 포폴)을 디자인 이슈로 오인 2건.

**총평**: 위계·카피·카드 시스템 이미 상당히 좋음(질문형 도구 카드·프로모 캐러셀·다음 할 일 래더).
발견은 마감 수준 4건 + 지연 UX 1건.
