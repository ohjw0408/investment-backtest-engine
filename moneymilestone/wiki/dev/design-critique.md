# F-6 디자인 크리틱 패스

방법 = 페이지당 6컷(데스크탑 1280/1920·모바일웹 390 × 라이트/다크) 육안 크리틱 → 오너 승인 → 구현.
앱 WebView는 390 근사(최종 실기기). 기준선 = 코인베이스 아키타입 + 토스급 밀도. 리스킨 금지.

## 배치 1: 홈 (2026-07-03) — 오너 승인 대기

**총평**: 위계·카피·카드 시스템 이미 상당히 좋음(질문형 도구 카드·프로모 캐러셀·다음 할 일 래더).
발견은 마감 수준 4건 + 지연 UX 1건.

| # | 발견 | 폼팩터 | 개선안 |
|---|---|---|---|
| D1 | 히어로 금액 `***,***,*** / 원` 두 줄로 깨짐 | 모바일 | 금액+원 nowrap + `clamp()` 폰트 자동 축소 |
| D2 | 지수 위젯 3열 그리드에 5항목 → 우하단 빈 구멍 | 데스크탑 | 마지막 행 균등 확장(grid auto-fit) 또는 3+2 균형 |
| D3 | 도구 카드 6개 세로 나열 → 모바일 홈 총길이 3,616px (데스크탑 2,567) | 모바일 | 모바일에서 2열 그리드로 압축(홈 ~40% 단축, 터치 타깃 유지) |
| D4 | 다크에서 위젯 내부 셀 구분선 거의 안 보임 | 다크 공통 | 다크 divider 알파 소폭 상승 |
| D5 | "다음 할 일"이 배당예정 API(콜드 10초+)를 기다렸다 전체 렌더 → 그동안 "불러오는 중" | 공통 | 즉시 렌더 가능한 행(리밸·은퇴) 먼저 표시, 배당 행은 도착 시 삽입 |

비이슈 판정: 위젯 타이틀 "calendar-check" = 테스트 계정의 커스텀 위젯명(실유저는 자기 이름) /
포폴 차트 스파이크 = 로컬 시드 데이터 특이.

스크린샷: 세션 scratchpad `f6_home/` (6컷 + 다크 재검 1) — 재촬영 절차는 이 문서 방법 참조.
