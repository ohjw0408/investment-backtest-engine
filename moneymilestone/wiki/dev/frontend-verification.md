# 프론트/디자인 변경 검증 규칙 (필수)

> 오너 지시(2026-06-19): **"모든 버튼 다 눌러보고 확인."**
> 디자인/UI/템플릿/JS를 바꾸면 "배선 확인"만으로 done 처리 금지 — 반드시 아래 절차를 거친다.

## 0. 왜 (배경)

- **"배선만 확인"은 검증이 아니다.** Jinja 컴파일 OK · `getElementById` 대상 존재 · HTTP 200 ·
  콘솔에러 0 — 이건 *코드가 로드된다*는 것뿐, *화면이 제대로 보이고 작동한다*는 보장이 아니다.
- 실제 사고(2026-06-19): myassets에 홈 디자인 이식 후 "배선 OK"만 보고 배포 →
  **종목 수정 모달이 투명하게 깨지고, 보유종목 칸막이·월별 상세 테두리가 사라짐.**
  근본 원인은 `static/css/style.css` 주석 안의 `*/` 시퀀스(`ds(--brand*/--ds-*)`)가
  블록 주석을 조기 종료시켜 직후 레거시 토큰 `:root{--card/--bg/--border...}`가 통째로
  파서에서 드롭된 것. **모달을 한 번이라도 실제로 열어봤으면 즉시 보였을 문제.**
- 교훈: **렌더 + 클릭 검증을 절대 건너뛰지 않는다.**

## 1. 환경 띄우기

```bash
# 서버 (백그라운드). .env의 FLASK_SECRET_KEY 사용
venv\Scripts\python.exe app.py

# 로그인 필요 화면 → dev 세션 쿠키 (로컬 전용, OAuth 우회)
venv\Scripts\python.exe tests\mint_session.py   # stdout 마지막 줄 = 쿠키
```

- ⚠️ `mint_session.py` stdout 첫 줄에 import-time Redis 경고가 섞임 → 쿠키는 **마지막 줄만**(`tail -1`).
- ⚠️ Windows = 쿠키 끝에 CR(`\r`) 붙음 → `tr -d '\r\n'`로 제거하고 써야 인증된다.
- ⚠️ Node/Playwright 스크립트는 **프로젝트 폴더 안**에서 실행해야 `node_modules`(playwright) 해석됨.
  Git-Bash의 `/tmp`는 Node에서 `C:\tmp`로 해석되니 쿠키·스샷 파일은 프로젝트 상대경로(`./_x`)로.

## 2. 데이터 시드 (빈 화면 금지)

테스트 유저에 데이터가 없으면 **실제 상태가 안 보인다**(빈 표/빈 탭). 검증 전에 시드한다.
- 한글 포함 POST는 셸 인코딩이 깨뜨림(0xb1) → **Python `urllib`로 UTF-8 JSON POST**.
- 로컬은 한국주식 시세 캐시가 없어 가격이 `None`(과거엔 `NaN`) → `/api/myassets/manual-price`로
  수동가격 넣어 보유종목·차트가 렌더되게 한다.
- myassets 리밸런싱/추가매수/그룹 검증 = 그룹 2개+목표비중+종목배정까지 시드해야 의미 있는 화면이 나온다.

## 3. 클릭/렌더 체크리스트 (변경 범위의 모든 항목)

```
□ 모든 버튼·링크를 실제로 클릭 → 기대 동작 확인
□ 모든 탭 전환 → 각 탭 내용이 렌더되는지
□ 토글·체크박스·라디오·슬라이더·드롭다운 → 값 변하고 UI 반영되는지
□ 모달/바텀시트/팝오버 → 열어서 배경·레이아웃·닫기버튼까지 (투명/깨짐 없는지)
□ 차트 → hover 툴팁·클릭(드릴다운)·기간 전환 동작
□ 드래그(차트 줌 등) → 동작 + 모바일 대안(날짜 입력 등) 존재 확인
□ 입력 검증 에러·빈 상태·로딩 상태 표시
□ 네이티브 alert()/confirm()/prompt()가 뜨지 않는지(브랜드 모달로 대체됐는지)
□ 라이트 + 다크 모드 둘 다 스샷 육안
□ 콘솔에러 0 (pageerror·console.error 캡처)
□ 변경 전후 기능 100% 보존 — 인터랙션 누락·회귀 없음
```

Playwright로 `page.on('pageerror')`·`page.on('console')` 캡처 + 인터랙션 `.click()` 후
`.screenshot()` 저장 → **스샷을 직접 눈으로 본다**(읽기 도구로 이미지 확인). 데이터 의존부만 오너 라이브 스팟.

## 4. done 기준

- 위 체크리스트 통과 + 라이트/다크 스샷 육안 OK + 콘솔에러 0.
- **깨진 것 발견 시 고치기 전에 먼저 오너에게 보고**(임의로 범위 키우지 말 것).
- 검증 끝나면 임시 파일(`_*.mjs`, 스샷, `_cookie.txt`) 정리하고 서버 종료.
