입력(가정) → 결정론적 미래 프로젝션 → 표/그래프. 빠르고 가볍게. 시트 대체.

잼투리 시트 입력 예: 과세여부(15% 원천)·인플레율(3%)·배당성장률(8%)·배당수익률(3.6%)·

## 후보 계산기 (오너 지정)

| 계산기 | 입력(가정) | 출력 | 비고 |
|---|---|---|---|
| **복리 계산기** | 초기금·월적립·연수익률·기간·(연증액) | 만기 평가액·원금·수익·연도별 추이 | 가장 기본 |
| **인플레이션 생활비 계산기** | 현재 월생활비·인플레율·기간 | N년 후 필요 월생활비(명목)·누적 | "지금 300만 = 20년 후 ?" |
| **실질 구매력 계산기** | 금액·인플레율·기간 | 미래 명목가치의 현재가치(실질) | 위와 짝 — 명목↔실질 |
| **배당금 계산기(재투자)** | 잼투리식: 과세여부·배당수익률·배당성장률·인플레·주기·초기/월투자·연증액 | 포트폴리오 추이·누적배당·세후·인플레조정 | ⚠️ 기존 배당 계산기(`dividend_target`/`dividend_simulator`)는 **롤링 실데이터 역산** — 이건 **가정 기반 프로젝션**으로 별개. 통합/구분 결정 필요 |

## 설계 방향 (미확정 ⏳)

- **공통 패턴:** 슬라이더/입력 → 즉시 클라이언트 계산(서버 불필요할 수도) or 가벼운 API.
- **세금:** 과세여부 토글(배당 15.4%/15%·면세) — 간단 적용. 무거운 엔진 안 씀.
- **인플레 조정:** 실질 = 명목 / (1+인플레)^년. 공통 유틸.
- **UI:** 탭? 별도 페이지 묶음? 모바일 우선(타겟층).
- **기존 배당 계산기와 관계:** 별도 신규 vs 기존에 "간편 모드" 추가 — 결정 필요.

## 구현 (2026-06-10, Claude)

- 신규 페이지 `/simple` (app.py 라우트 + base.html nav/sidebar "간편 계산기" 링크)
- `templates/simple.html` — 탭 4개(복리·배당재투자·인플레 생활비·실질 구매력), 입력 변경 즉시 재계산
- `static/js/simple_tools.js` — 순수 계산 함수 4종 + 렌더/탭. 서버 호출 없음.
- 계산 규약: 월초 적립 → 월복리(월율=연율 기하환산). 과세=15.4% (복리=세후 수익률, 배당=배당마다 원천징수 후 재투자). 실질가치=명목/(1+인플레)^년. 연증액률=매년 월적립 ×(1+g).
- 배당 모델: 주가는 배당성장률로 상승(시가배당률 일정), 주기(분기/월배당)마다 평가액×수익률/횟수 지급 → 세후 전액 재투자 (스노우볼).
- 복리·배당 탭은 Chart.js 라인차트(명목·실질·원금/누적배당) + 연도별 표. 인플레·구매력은 표.
