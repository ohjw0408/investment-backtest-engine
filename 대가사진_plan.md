# 투자대가 초상 사진 도입 — 플랜

## 배경 / 결정(2026-06-25)
현재 대가 아바타 = 제너러티브 추상 도형(`guru_avatar.js`). 오너: **실제 사진 넣고 싶다.**

법적 재평가 결론(오너 동의):
- 진짜 챙길 리스크 = **저작권**(사진작가) → **CC/PD 사진만 + 출처표기**로 클리어.
- 퍼블리시티권/초상권(한국 부정경쟁방지법 타목)은 이론상 0 아님이나 **실무 분쟁 사실상 전무**(유튜브 썸네일·금융미디어 광범위 사용). 정보 맥락(13F 공개정보) + 출처표기 + 면책으로 방어선. **감수하고 진행.**
- AI 생성 초상 = 안 씀(애매·식별 리스크 동일).

## 방향 = 혼합(사진 + 추상 폴백)
- **CC/PD 무료사진 확보된 6명** → 사진.
- **무료사진 없는 4명** → 현행 추상 아바타 유지(빈칸 X). 나중에 사진 생기면 교체.

## 작업 단계
- **P1 라이선스 실사**: 6명 각 파일의 정확한 라이선스(PD / CC BY / CC BY-SA + 버전) + 저작자명 + 원본 URL 확정. 표로. ⚠️ CC BY-SA = **저작자 표시 의무**(안 하면 저작권 위반). 표기 불가/모호하면 그 사람도 추상으로.
- **P2 다운로드·호스팅**: 핫링크 금지(커먼스 정책). `static/images/gurus/<slug>.jpg`로 다운로드(적정 해상도 리사이즈, ~400px 정사각 크롭). 출처 메타는 코드/JSON에.
- **P3 데이터**: 사진+크레딧을 어디 둘지 — guru_holdings.db 스키마 변경 대신 **코드 매핑**(store.py `_PHOTOS = {slug: {file, credit, license, source_url}}`)이 간단(seed DB 재빌드 불요). store가 `photo`(url)·`photo_credit` 노출.
- **P4 렌더**: 아바타 = 사진 있으면 `<img>` 원형 크롭(object-fit:cover), 없으면 현행 `data-gavatar` 추상. gurus 카드·상세·홈 프로모 공통. guru_avatar.js 또는 매크로 분기.
- **P5 출처/면책**: 상세 페이지에 사진 출처·라이선스 캡션(예: "사진: ⓒ작가 / CC BY-SA 4.0 / Wikimedia"). 약관/면책 한 줄("공개 정보·인물 사진 출처 표기, 본 서비스는 해당 인물과 무관").
- **P6 검증**: Playwright 라이트/다크 — 6명 사진 원형 렌더·4명 추상 폴백·크레딧 캡션·깨짐 없음. 라이선스 표 최종 점검.

## 주의 / 디자인
- 사진 구도 제각각(행사컷·측면) → 정사각 크롭 시 얼굴 중심 안 맞을 수 있음. 크롭 좌표 개별 조정 or object-position.
- 사진/추상 혼합이라 **원형 틀**로 통일(현재 아바타와 동일 크기·테두리).
- 홈 프로모(마케팅성)도 사진 쓸지 vs 추상 유지 → **P4서 결정**(보수적이면 프로모는 추상, 정보페이지만 사진). 일단 전부 사진으로 가되 어색하면 프로모만 추상.

## 미결(진행 시 확인)
- 6명 중 라이선스 모호한 사람 있으면 추상으로 강등.
- 크레딧 표기 위치(상세 캡션 필수, 카드·프로모는 생략 가능한지 — CC BY-SA는 "합리적 방식" 표기면 OK, 보통 상세/about에 모아도 됨).
