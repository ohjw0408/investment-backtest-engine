## 2.5 ★ 방향 수정 (2026-06-18) — 아키타입 기반 전면 재구성

**원인:** 일관성을 색/토큰에서만 찾고, **레이아웃·여백·위계·상태**를 안 건드림.
페이지마다 목업 만드는 건 비현실(24장). 일관성의 진짜 출처 = **아키타입(공통 레이아웃 패턴)**.

## 3. 전략 — 3층 아키텍처 (이게 "통일"을 만드는 핵심)

```
┌─ 3층: 페이지 (24개 템플릿)  ← 인라인/page-style 제거, 공용 클래스만 사용
├─ 2층: 공용 컴포넌트 (components.css + JS 헬퍼)  ← 버튼·카드·입력·모달·토스트·테이블…
└─ 1층: 디자인 토큰 (style.css :root / [data-theme])  ← 색·폰트·간격·라운드·타입
```

원리: **페이지는 토큰을 직접 쓰지 않고 컴포넌트를 쓴다.** 컴포넌트가 토큰을 참조.
→ 색 하나 바꾸면 토큰만, 버튼 모양 바꾸면 컴포넌트만. 페이지는 안 건드림.
이게 산재(인라인 732)가 다시 안 생기게 막는 구조적 방벽.

## 5. 공용 컴포넌트 카탈로그 (2층 — 한 번 만들고 전 페이지 재사용)

각 컴포넌트 = 라이트+다크 + 모든 상태 동시 정의.

- **버튼**: primary(주황+ink) · hover · secondary(회색) · ghost · text · disabled · pill CTA(큰) · 아이콘버튼
- **카드**: 기본(헤어라인+24px) · hover · 다크밴드(에디토리얼) · 섹션헤더
- **입력**: text · number · select · 검색 pill · 포커스(주황 2px) · 검증에러(빨강+메시지) · disabled · 라벨/헬퍼텍스트
- **탭/세그먼트**: 기간 pill · 탭바 · 활성표시
- **테이블/리스트**: 자산행(원형아이콘+모노숫자) · 정렬헤더 · 등락 셀(녹/적) · 모바일 카드 변환
- **모달·바텀시트**: 오버레이 · 헤더/바디/푸터 · 모바일 바텀시트 · 닫기
- **토스트**: 성공·에러·정보 (← `alert()` 60곳 교체)
- **확인 다이얼로그**: (← `confirm()` 11곳 교체)
- **상태 표시**: 로딩 스피너/스켈레톤 · 빈 상태(아이콘+문구+CTA) · 에러 상태
- **배지·칩·툴팁·포popover**: (배당 팝오버 등 기존 것 흡수)
- **차트 래퍼**: Chart.js 색을 토큰에 바인딩(전 차트 일관 색)

## 5.5 액센트 색 선택 (테마 컬러)

유저가 voltage 색을 ~10색 중 고름. **기본 = 블루**(신규유저·비로그인·마케팅). **로고는 액센트와 무관하게 항상 주황**(브랜드 마크 — voltage 아님).

**왜 싸게 되나:** 3층 구조라 전 페이지가 `--brand*` 토큰만 참조. 액센트 = 그 토큰 묶음만 교체 → 페이지/컴포넌트 무수정.

**메커니즘:**
- `<html data-accent="orange">` 속성으로 스왑. CSS `[data-accent="blue"]{ --brand:…; --brand-hover:…; --brand-tint:…; --brand-text:…; --on-brand:…; }` (라이트), `[data-theme="dark"][data-accent="blue"]{…}` (다크).
- 저장: `localStorage('mm-accent')` (테마 `mm-theme`와 동일 패턴). base.html FOUC 방지 인라인 스크립트에서 CSS 로드 전 `data-accent` 설정.
- 로그인 유저는 프로필에도 저장(선택, 기기간 동기화) — localStorage가 1차.
- UI: **설정 페이지에 스와치 그리드**(원형 칩 10개, 선택 표시). 라이트/다크 토글과 나란히.

**색 팔레트 (~10, 각 튜닝셋 필요):**

| 이름 | fill(`--brand`) | 비고 |
|---|---|---|
| 🟦 블루 Blue | `#0052ff` | **기본** · 코인베이스 |
| 🟧 주황 Orange | `#ff6a00` | 로고색과 동일(원하면 통일 가능) |
| 🩵 스카이 Sky | `#0ea5e9` | |

| ⬛ 그래파이트 Graphite | `#475569` | 중성·미니멀 |

**제약:**
- **녹색(`--up`)·빨강(`--down`)·금(`--gold`)은 시맨틱 예약** → 액센트 후보서 제외(등락·금과 혼동 방지). 웜 계열은 주황 1개만(나머지 쿨).
- **로고 주황은 액센트 토큰과 분리** — 별도 고정 변수(`--logo`, 항상 `#ff6a00`). 액센트 스왑이 로고를 안 건드리게.
- 각 색마다 튜닝 의무: `--brand-text`=흰 배경 가독 위해 어둡게 / `--on-brand`=fill 위 글씨를 ink·white 중 4.5:1 통과하는 쪽 / 다크 variant=톤업. 구현 시 색당 5값×(라이트+다크) 산출.
- 차트·스파크라인·활성표시 전부 `--brand` 참조라 액센트 바꾸면 자동 따라감.

## 7. 페이지 작업 방법 — 페이지마다 5단계 (A~E)

### A. 감사 (Audit) — 코드 정독 + 실제 구동
정적 템플릿 + **JS 정독** + **앱 실제 실행해 상태 열어보기**. 아래를 빠짐없이 목록화:

```
□ 입력 요소: 모든 text/number/select/checkbox/radio/슬라이더/날짜/드래그
□ 컨트롤: 모든 버튼·링크·탭·토글·드롭다운·아코디언
□ 동적 UI: innerHTML로 그려지는 모든 것(탭 전환 결과, 리스트, 차트, 그리드)
□ 조건부 표시: ~일 때만 나오는 영역(로그인/비로그인, 데이터유무, 멀티계좌 등)
□ 상태: 로딩 · 빈("없습니다") · 에러 · 검증실패 메시지 · 성공
□ 오버레이: 모달·바텀시트·팝오버·툴팁·드롭다운 메뉴
□ 네이티브 교체: 이 페이지의 alert()/confirm() 위치 → 토스트/다이얼로그로
□ 반응형: 모바일에서 바뀌는 레이아웃·숨는 것·드래그 대안
□ 다크모드: 위 전부의 다크 상태
```
→ 산출물: 페이지별 감사 노트(`moneymilestone/wiki/dev/` 또는 이 plan에 append).

### B. UX 재설계 제안
감사 결과 토대로 "더 쉽게" 개선안 작성 → **오너 확인**(아키타입 대표 페이지만, 동류는 패턴 재사용).

### C. 구현
공용 컴포넌트로 마크업 교체. 인라인 style 제거. page `<style>` 블록 → 공용 또는 페이지 전용 최소 CSS. JS의 `.style`/`innerHTML` 생성 마크업도 새 클래스로.

### D. 전 상태 다크모드 마감
감사에서 목록화한 모든 상태를 라이트+다크 둘 다 확인.

### E. 검증
- 내가 **서버 백그라운드 실행 + Playwright로 상태별 스샷** → 오너 육안
- 타겟 테스트만(전체 pytest 금지 — CLAUDE.md): UI는 jsdom/Playwright, 변경 API는 해당 테스트
- 기능 보존 확인: 감사 목록의 모든 인터랙션 동작 재확인

## 8. 미리보기·검증 워크플로 (오너 로컬 구동 불가 대응)

1. **내가 이 환경에서 Flask 서버 백그라운드 실행** + Playwright(설치됨)로 페이지/상태 스샷 → 오너에게 제시.

3. **프로덕션 푸시 = 배치 승인 후에만.** 반쪽 상태 프로덕션 금지(지난 실수 재발 방지).
4. 푸시 단위 = 배치(파운데이션 / 아키타입별). 문제 시 배치 단위 revert.
