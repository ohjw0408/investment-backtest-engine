## 한 줄 정의

저장(즐겨찾기)한 포트폴리오들을 **x축=연평균 변동성, y축=연평균 수익률** 산점도에 찍고,
대표지수/자산(QQQ·SPY·GLD 등)과 같은 평면에서 비교. "내 포폴이 위험 대비 수익 어디쯤?"

## 선행조건 — 포트폴리오 즐겨찾기

- 사용자가 구성한 포트폴리오를 저장/이름붙임/불러오기. **미구현.**
- 이 도표는 즐겨찾기된 N개 포폴을 점으로 비교 → 즐겨찾기 먼저.

## 데이터 / 지표 (엔진 이미 보유)

- **연평균 수익률(y):** CAGR 또는 연율화 수익률. `calc_metrics_from_history`의 `cagr`.
- **연평균 변동성(x):** 일간수익률 std × √252 (연율화). 현재 sharpe 계산에 std 이미 씀 → 노출만.
- 각 포폴: 같은 기간·같은 데이터로 두 지표 산출 → 점 (x, y).

## 벤치마크 (비교 대상)

- 대표 자산 QQQ·SPY·GLD(+코스피·미국채 등) 가격 로드 → **동일 기간** 같은 지표 산출 → 점.
- 사용자 포폴과 같은 롤링/기간 정렬 필수(공정 비교).
- ⚠️ 기간 정렬: 포폴마다 데이터 시작 다르면? → 공통 겹치는 기간으로 맞추거나, 각자 전체기간(주의 표기).

## UI

- 산점도(Chart.js scatter). 내 포폴 = 강조색, 벤치마크 = 회색/다른 마커.
- 호버: 이름·CAGR·변동성·샤프. 대각선(샤프 등고선) 옵션.
- 모바일 우선(타겟층).

## 결정 대기 (오너)

1. **선행:** 즐겨찾기 기능 스펙(저장 위치·서버 vs 로컬·개수 제한).
2. 수익률 정의: CAGR vs 연율 산술평균 vs MWR(적립 포함 시).
3. 기간 정렬 정책(공통기간 vs 전체기간).
4. 벤치마크 목록 고정 셋 vs 사용자 추가.
5. 변동성 = 일간 기반 √252 vs 월간 기반.

## 검증

- 지표 산출은 이미 `calc_metrics`에서 검증됨(테스트 존재). 도표는 그 값 시각화 → 저위험.
- 벤치마크 지표가 알려진 값(예: SPY 장기 변동성 ~15%대)과 합리적 범위인지 sanity 체크.

## 우선순위

선행(즐겨찾기) 없이는 불가 → **즐겨찾기 구현 후**. 그 전엔 아이디어 동결.
