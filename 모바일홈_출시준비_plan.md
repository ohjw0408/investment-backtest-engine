## 0. 정체성 (모든 결정의 기준)

- **이름:** Money Milestone = "당신의 돈의 이정표" → 잔고 스냅샷 앱 ❌, **여정(journey) 앱** ✅.
- **페르소나:** **자산배분 투자자** (분산 포폴·리밸런싱·장기계획). 데이트레이더 아님.
- **밀고 싶은 3기능 (= 3기둥):**
  1. **자산 관리** — 추이·수익·비중 (엑셀로 매달 긁던 고통 해소 = 앱의 실용 가치)
  2. **리밸런싱 알림** — 앱의 **오리진**. "균형 깨지면 알려줌." (획득용 아님 = 잔존용)
  3. **계산·시뮬레이션** — 백테·은퇴·배당·계산기·비교 (로그인 없이 = **획득 후크**)
- 부가(중요도 낮음): 거시지표·지수·(추후 S&P 히트맵) → "시장" 하단 접힘.
- 역할 분담: **획득 = 기둥3(로그인 없이 시뮬), 잔존 = 기둥2(알림·이정표).**

## 1. 앱 vs 웹 분기 (질문 답 + 전제)

**가능하다.** `html.is-app` 클래스가 Capacitor 네이티브 감지 시 붙음 (`base.html:71, 303`).
- 앱에서만: `html.is-app .x { }`
- 웹에서만: `html:not(.is-app) .x { }`
- ⚠️ 서버는 앱/웹 구분 못 함(같은 WebView UA, overrideUserAgent까지 동일) → **분기는 클라이언트 CSS/JS의 `is-app`로만.** Jinja `{% if %}` 불가.

→ **다운로드 QR/배너 = `html:not(.is-app)`** 로 모바일웹·데스크톱웹에만, 앱에선 자동 숨김.

## 2. 모바일 홈 재구성 (비로그인, 모바일 ≤768 / 앱만 — 데스크톱 불변)

### 2.1 원칙
- **압축 3도어(door) 카드** — 각 기둥을 풀 콘텐츠 대신 *한 줄 요약 + 미니 비주얼 + 화살표*로 압축 → 390폭 첫 화면(fold)에 3장 다 보임. 누르면 해당 기능 진입.
- **로그인 CTA 카드 제거** (오너: 로그인 유도는 딴 탭서 충분). 로그인은 각 기능 진입 시 그 페이지가 처리.
- 데스크톱·로그인 후 홈은 **이번 범위 아님**(추후 로그인 홈=이정표 진행도로 진화).

### 2.2 레이아웃 (390폭)
```

│ ☰   Money Milestone    🌙 🔔  │  nav
├──────────────────────────────┤
│ 자산배분 투자자를 위한 올인원   │  히어로(슬림 ~56px)
├──────────────────────────────┤
│ 📈 자산 관리              ›   │  기둥1 (~150px)
```

- 기둥3 = 기존 6엔진(`feat-grid`) 재사용, 아이콘 줄로 압축.

### 2.3 구현
- `templates/index.html` 비로그인 분기 재구성. **모바일/앱만** 새 압축 레이아웃, 데스크톱은 기존 유지(미디어쿼리 + 필요 시 `is-app`).
- 기존 `feat-grid`·`demoChart`·CSS `order` 패턴 재사용. 기존 ID/JS 보존.
- 시장지수 위젯 = 하단 접힘.

### 2.4 검증
- Playwright 모바일 390 비로그인: 3도어 fold 위 노출·오버플로우0·라이트/다크·콘솔0. 데스크톱 회귀0.

## 3. 첫 실행 스와이프 튜토리얼 (온보딩)

### 3.1 역할
정체성 **스토리텔링**(홈은 런처라 설명 부담 덜어줌). 첫 실행 1회 자동, 이후 숨김.

### 3.2 카드 4장 (글 + 미니 UI 목업 — **이미지 에셋 의존 0**, 디자인 결 일치)
1. **엑셀, 이제 그만** — 자산 자동 추적 (자산관리)
2. **균형 깨지면 알려드려요** — 리밸런싱 알림 (오리진)
3. **30년 미래를 미리 돌려봐요** — 계산·백테·은퇴·배당
4. **"당신의 돈의 이정표"** + [시작하기] (정체성 + 진입)

### 3.3 동작
- 첫 실행 자동 (`localStorage: mm_onboarded`), **건너뛰기·도트 인디케이터·스와이프(터치)**, 마지막 [시작하기].
- 재열기: 우상단 `?` 또는 설정. 다신 자동 안 뜸.
- 순수 웹 오버레이 → 앱(원격로드)서 그대로, **앱 재빌드 0.**
- 앱 우선(`is-app`)이되 모바일웹서도 노출 가능(결정: 일단 앱+모바일웹 공통).

### 3.4 구현
- `base.html` 또는 신규 `static/js/onboarding.js` + 인라인 CSS. 오버레이 컴포넌트(`mmToast`/`mmConfirm` 결).
- 검증: Playwright 첫 진입 자동 노출·스와이프·건너뛰기·재진입 안뜸·라이트/다크.

## 4. 앱 다운로드 프로모 (웹 전용)

### 4.1 목표
모바일웹·데스크톱웹 방문자를 **Play Store 앱 설치**로. 앱 내부(`is-app`)선 숨김.

### 4.2 구현
- `html:not(.is-app)` 로만 노출.
- **데스크톱웹:** 우측/하단 카드 — Play 배지 버튼 + **QR코드**(폰 카메라로 설치).
- **모바일웹:** 상단/하단 슬림 배너 — "앱으로 더 편하게" + [Play스토어] 버튼(딥링크).
- QR = 정적 PNG 에셋 또는 `qrcode` 라이브러리 클라 생성(Play URL).
- Play URL: `https://play.google.com/store/apps/details?id=com.moneymilestone.app` (출시 후 확정).
- 닫기(localStorage `mm_dl_dismiss`)로 재노출 억제.

### 4.3 검증
- 웹(`is-app` 없음)=배너 보임 / 앱(`is-app`)=숨김. Playwright 양쪽 토글.

## 5. 스플래시 · 로고 애니메이션 고급화

### 5.1 현황
- 웹 스플래시 애니메이션 **없음**. 지금 "로고 확대"=Android 12+ 시스템 기본 스플래시(아이콘+시스템 줌, 제한적).
- 네이티브 첫 프레임은 OS가 그림 → 커스텀 애니 한계.

### 5.2 방안 (2층)
- **(A) 웹 오버레이 스플래시 (메인, 앱 재빌드 0):** WebView 부팅 시 풀스크린 오버레이 → **SVG 로고 리빌(stroke draw-in) + 부드러운 ease + 브랜드 그라데/은은한 shimmer + "이정표" 모션**(점→선 잇는 여정 메타포). 앱 준비되면 페이드아웃. `is-app`서만(웹은 즉시 콘텐츠).
  - 금융앱 신뢰 = **절제된 고급**(과한 모션❌, 0.6~1.0s, 부드러운 감속, 모노 느낌).
  - 구현: `base.html` 인라인 SVG+CSS `@keyframes` + JS(앱 로드 완료 시 제거). 라이트/다크 대응.
- **(B) 네이티브 스플래시 정적 프레임 정돈 (선택, APK 재빌드 필요):** Android 12 splash API — 단색 배경 + 고해상 모노크롬 아이콘 정렬. `@capacitor/splash-screen` 설치 시 제어. 애니는 (A)가 담당, (B)는 첫 프레임 품질만.

### 5.3 검증
- 앱: 부팅 오버레이 노출→페이드아웃·콘솔0. 웹: 미노출(즉시 콘텐츠). 라이트/다크. (실기기 최종 육안=오너.)

## 6. Play Store 출시 에셋

### 6.1 필수 에셋 체크리스트
- [ ] **앱 아이콘** 512×512 PNG (32-bit, alpha). (별도 adaptive 1024 = 메모리 LAUNCH 잔여)
- [ ] **피처 그래픽** 1024×500 PNG/JPG (no alpha) — 상단 배너.
- [ ] **폰 스크린샷** 2~8장, 9:16 (예 1080×1920) PNG/JPG.
- [ ] (선택) 7"·10" 태블릿 스크린샷.
- [ ] **짧은 설명** ≤80자.
- [ ] **자세한 설명** ≤4000자.
- [ ] **개인정보처리방침 URL** — `/privacy` 보유 ✅.
- [ ] 데이터 안전(Data safety) 양식, 콘텐츠 등급 설문, 카테고리(금융), 연락 이메일.

### 6.2 스크린샷 = 코드로 자동 생성 (핵심 효율)
- **Playwright로 실제 앱 화면 1080×1920 캡처** → 상단 캡션 + 폰 프레임 합성.
- 6컷 제안 (= 3기둥 + 핵심):
  1. 3도어 홈 — "올인원 자산배분 앱"
  2. 내 자산 추이·비중 — "엑셀 없이 한눈에"
  3. 리밸런싱 알림 — "균형 깨지면 알려드려요"
  4. 백테스트 결과 — "30년 데이터로 검증"
  5. 은퇴/배당 시뮬 — "언제 은퇴 가능한지 계산"
  6. 거시·캘린더 — 부가
- 캡션 오버레이·프레임 = 스크립트(`scripts/store_shots.py` 또는 mjs)로 반복생성.

### 6.3 설명문 초안 (정체성 반영)
- **짧은 설명(≤80):** "자산배분 투자자를 위한 올인원 — 자산관리·리밸런싱 알림·30년 시뮬레이션"
- **자세한 설명(초안, 다듬기):**
  - 후크: "엑셀로 매달 가격 긁어 자산 계산하던 시간, 이제 끝."
  - 3기둥 소제목(자산관리/리밸런싱 알림/계산·시뮬) 각 2~3줄.
  - 차별점: "남들은 잔고만 보여주지만, 우린 *언제 은퇴 가능한지·세금 얼마 아끼는지* 계산합니다."
  - 신뢰: 시세 15분 지연 고지·투자자문 아님 고지.
  - CTA: "로그인 없이 지금 30년 시뮬레이션."

### 6.4 검증/산출
- 에셋 폴더 `store/` (gitignore 여부 결정) 또는 `moneymilestone/store-assets/`.
- 스크린샷 스크립트 결정론 캡처 + 육안.
