## 목표

결과 화면에 **3개 숫자**(계좌별 + 합산):
1. **전체 위탁 가정 세금** — 모든 계좌가 위탁이었다면 냈을 세금(분석적 추정).
2. **실제 세금** — 현재 구성으로 낸 세금(엔진이 이미 계산).
3. **절세액** = ① − ②.

## 2. 추적 메커니즘 (per-account, 세전 누적 — 신규)

위탁 가정 세금을 역산하려면 계좌 유형과 **무관하게** 세전 흐름을 누적해야 한다.
(ISA/연금은 비과세라 현재 미추적 → 추적 추가.)

계좌별 신규 누적 필드:
- `cf_gross_dividend_by_class` — 자산분류별 **세전 배당** 누계(전 기간).
- `cf_kr_foreign_gain` — KR_FOREIGN **실현+미실현 차익** 누계.
- `cf_us_gain_by_year` — US_DIRECT 양도차익을 **연도별** 누적(연 250만 공제·절세매도 정확성에 필수).
- (국내·KRX금 = 0이라 추적 불필요.)

### 2-1. ISA 풍차 누적 (오너 질문 — "트래킹 가능한가")
가능. 풍차 만기마다 ISA가 **전량 청산**되므로, 그 시점이 위탁 가정의 "실현 이벤트":
- 각 만기에서 그 사이클 차익(`V − cycle_contribution`)을 자산분류별로 위 누적에 가산.
- 배당도 사이클 무관 전 기간 `cf_gross_dividend_by_class`에 누적.
- 최종(미만기) 사이클은 종료 시 미실현분 가산.
→ **풍차 전 사이클 통합 추적** → 위탁 가정 = "풍차 없이 위탁으로 12년 굴렸으면" 누적 세금.
- `_mature_isa`에 누적 훅 추가(L-SAVE2로 검증).

### 2-2. 절세매도(gain_harvesting) (오너 질문)
GH on 위탁계좌는 매년 US 차익을 실현해 **연 250만 공제를 매년 소진**.
- `cf_us_gain_by_year`가 **연도별**이므로 위탁 가정 세금 = `Σ_year max(0, us_gain_year − 250만)×0.22`
  → GH on이면 연도별로 쪼개져 250만/년 공제 자동 반영, GH off면 청산연도에 몰려 단일 250만 공제.
  **같은 연도별 자료구조로 두 경우 자연 처리.**
- 위탁 가정은 **각 계좌의 실제 실현 패턴(GH 설정·연도별 실현분)을 그대로 사용**(재가정 안 함).
  ⇒ ISA→위탁 가정 시 ISA는 청산 단일실현(단일 250만 공제), GH 켠 위탁계좌는 연도별. (오너 검토)

## 3. 위탁 가정 세금 — 순수함수 (공통 모듈)

`modules/tax/saving_estimate.py` 신규:
```
estimate_brokerage_tax(gross_div_by_class, kr_foreign_gain, us_gain_by_year) -> float:
    배당세   = Σ_class gross_div_class × 배당세율(class)
    krf_세   = max(0, kr_foreign_gain) × 0.154
    us_세    = Σ_year max(0, us_gain_year − 2_500_000) × 0.22
    # 국내·KRX금 = 0
    return 배당세 + krf_세 + us_세
```
- 순수함수 → 단위 테스트(L-SAVE0) 직접.
- 전 탭(P1~P3) 재사용.

### 계좌별 절세액
- `절세액 = max(0, 위탁가정세금 − 실제세금)` (0 하한 — 손실·비과세내 음수 방지).
- 위탁 계좌: 위탁가정 = 실제 → 절세 0 (불변식).

## 4. 기준값 (오너 질문 — "중앙값? 뭘로?")

롤링은 윈도우(케이스)마다 절세액이 다름. **결정: 케이스별 절세액 → 중앙값(p50) 표시.**
- 이유: end_value 중앙값 케이스의 절세액(g2 대표케이스)은 절세액 분포의 대표가 아닐 수 있음.
  절세액 자체의 p50이 "전형적 절세"를 더 잘 나타냄.
- 보조로 평균·범위(p25~p75) 함께 표기 가능.
- 합산 절세액도 케이스별 Σ계좌 → 그 분포의 p50. (단일계좌 회귀와 일관.)
- (대안: g2 대표케이스 기준 — 일관성↑이나 대표성↓. **p50 채택.**)

|---|---|---|---|

## 6. 검증 설계 (L-SAVE — 결정론 손계산, **최우선**)

> 모든 케이스 결정론 픽스처(계단/단일배당). 정상경로 손계산 ±1원 + 경계 + 불변식.

| 계층 | 대상 | 손계산 검증 | 경계/엣지 |
|---|---|---|---|
| **L-SAVE0** | 순수함수 `estimate_brokerage_tax` | 배당 X·KRF Y·US(2년 각 Z)×공제 → 세금 정확 | US 250만 공제 정확·손실(0)·금0·0입력 |
| **L-SAVE1** | ISA 단일사이클 절세 | 계단가격: ISA 순익1천만−비과세200만→실제 79.2만, 위탁가정=배당15.4%+양도15.4%, 절세=차이 | ISA 비과세내(실제0, 절세=위탁가정)·손실(절세0) |
| **L-SAVE2** | **ISA 풍차 다중사이클 누적** | 9년 3사이클 ×2: 각 만기 실현차익 누적→위탁가정(만기별 양도세 합) vs 실제(만기세 누적), 절세=차이 ±1원 | 마지막 미만기 사이클 미실현 가산·풍차중단(금종세) 시 누적 일관 |
| **L-SAVE3** | **절세매도(GH) 연도별 공제** | 위탁 GH on, US 차익 매년 300만씩 2년 → 연도별 max(0,300−250)×22%×2 vs GH off 단일(600−250)×22% → 위탁가정이 연도별 반영 | GH on/off 세금 차이 정확, 250만/년 경계 |
| **L-SAVE4** | 연금 수령세 vs 위탁가정 | 나이 기반 연금소득세 손계산 vs 위탁가정, 절세 부호·크기 | (1500만 한도는 P3) |
| **L-SAVE5** | 위탁 불변식 | 위탁 계좌 절세액 == 0 | 항상 |
| **L-SAVE6** | KRX금 | 금 차익 → 위탁가정 0, 실제 0, 절세 0 | 금 단독·금+주식 혼합 |
| **L-SAVE7** | 합산 = Σ계좌 | combined 절세액 == Σ account 절세액 (케이스별) | |
| **L-SAVE8** | **기준값(p50)** | 손계산 가능한 2케이스(절세 A·B) → 표시값 = median(A,B) | 케이스 1개·홀짝 |

각 단계 P1~P3는 해당 L-SAVE 통과 후 다음 탭. **L-SAVE2·3·8이 네가 짚은 난점(풍차·절세매도·기준값) — 별도 결정론 테스트로 못 박음.**

### 전 케이스 불변식
- 절세액 ≥ 0 (0 하한 가드).
- 위탁 절세액 = 0.
- 위탁가정세금 ≥ 실제세금 (세제혜택계좌는 항상 이득 — 단 종합과세 가산 생략으로 드문 반례 가능, 명시).

## 7. 결정 사항 (확정/대기)
- ✅ KRX금 0%.
- ✅ 위탁 가정은 각 계좌 실제 실현패턴(연도별·GH설정) 그대로 사용.
- ✅ 기준값 = 케이스별 절세액의 **p50(중앙값)**.
- ✅ 종합과세 가산 생략(러프) — 시작점. 필요시 후속.
- ⏳ 연금 수령세 러프 산식 = `TaxEngine.after_tax_withdrawal` 연금경로 재사용(P1서 확정).
- ⏳ 1,500만 수령한도 초과세 = P3(은퇴)에서만.
