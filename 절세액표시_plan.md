# 절세액 표시 기능 계획

작성: 2026-06-02 (Claude, 오너 요청)
연관: Track G 멀티계좌(`trackG_multiaccount_plan.md`) — 계좌별 세금이 이미 계산됨.

---

## 목표

결과 화면에 **3개 숫자** 표시(계좌별 + 합산):
1. **전체 위탁 가정 세금** — 모든 계좌가 위탁이었다면 냈을 세금.
2. **실제 세금** — 현재 계좌 구성으로 낸 세금.
3. **절세액** = ① − ②.

세제혜택계좌(ISA·연금·IRP)의 가치를 "얼마 아꼈나"로 직관적으로 보여줌(마케팅 강점).

---

## 계산 방식 — 분석적 추정 (오너 결정: "러프하게")

재시뮬(전부 위탁으로 한 번 더 돌리기)은 롤링 2배 비용이라 **불채택.**
대신 **각 계좌가 이미 가진 세전 흐름에 위탁 세율을 적용**해 "위탁 가정 세금"을 역산. 1× 비용.

> ⚠️ 러프함의 근거: ISA는 배당을 net(세후)으로 재투자하므로 위탁(net 다름)과 holdings가
> 미세하게 다를 수 있음. 절세액은 **근사치**임을 UI에 명시(예: "약 ₩…").

### 위탁 가정 세금 = 세전 흐름 × 위탁 세율
- **배당**: 국내 15.4% / 미국직상장 15% / KR_FOREIGN(국내상장 해외ETF) 15.4%.
- **양도차익(청산)**: KR_FOREIGN 15.4% / US_DIRECT 22%(250만 공제) / 국내 0% / KRX금 22%.
  (종합과세 가산은 러프 단계에선 생략 — 임계 초과 시 과소추정 가능, 명시.)

### 계좌별 절세액
- **ISA**: 위탁가정(배당+양도) − 실제(만기 9.9%, 비과세 200/400만). 손실/비과세내면 절세 0~음수 → 0 하한 가드.
- **연금/IRP**: 위탁가정 − 실제(연금소득세 3.3~5.5%, 나이/수령개시나이 기반 러프).
  - **연 1,500만 수령한도 초과세(16.5%/종합)는 이번 단계 무시** — 은퇴 탭에서만 고려(오너 결정).
- **위탁**: 절세액 0(동일). 불변식.

## 엔진이 노출해야 할 것 (계좌별 — 신규)

현재 `account_result`에 있음: `tax_paid`(실제 합계)·`dividend_tax_paid`·`realized_tax_paid`·
`liquidation_tax_paid`·`maturity_tax_paid`·`kr_foreign_unrealized_gain`·`raw_end_value`·`total_contribution`.

**추가 필요(위탁 가정 계산용 — 세전, 계좌유형 무관):**
- `gross_dividend_total` — 계좌가 받은 세전 배당 누계. (현재 ISA/연금은 배당 비과세라 미추적 → 추적 추가.)
- `gain_by_class` — 청산 시 자산분류별 차익: `{KR_FOREIGN, US_DIRECT, KR_DOMESTIC, KRX_GOLD}` 실현+미실현.

이 둘이면 위탁 세율 적용해 "위탁 가정 세금" 역산 가능.

## 탭별 배선 (복잡도 순 — 단계)

| 단계 | 탭 | 복잡도 | 비고 |
|---|---|---|---|
| **P1** | 투자계산기 | 중 | 멀티계좌 loop·analyzer에 세전흐름 노출 + 위탁가정 계산 + 결과 패널. **여기서 방식 확정.** |
| **P2** | 백테스트 | 하(가장 단순) | 단일기간, P1 계산 모듈 재사용. |
| **P3** | 은퇴 | 상(복잡) | 인출기 + **연 1,500만 수령한도 초과세** 추가 고려 + 수령나이. P1 모듈 확장. |
| **P4** | 배당금계산기 | (보류) | 별도 엔진(`DividendSimulator`), 전면 재작업 필요 → 우선순위 낮음, 후속. |

**공통 모듈화:** 위탁 가정 세금 계산을 `modules/tax/`에 순수함수로 추출
(`estimate_taxable_account_tax(gross_dividends, gain_by_class, ytd_us_gains)`) → 전 탭 재사용 + 단위 테스트.

## 검증 설계 (L계층 스타일 — 결정론 손계산)

### L-SAVE0 — 위탁 가정 세금 순수함수 (단위)
- 계단가격 픽스처: 배당 X원·KR_FOREIGN 차익 Y원·US 차익 Z원 → 위탁세금 = X·0.154 + Y·0.154 + max(0,Z−250만)·0.22 ±1원.
- 경계: 손실(0), US 250만 공제 정확, 0 입력.

### L-SAVE1 — ISA 절세액 (투자계산기, 결정론)
- 계단가격: ISA 순이익 1천만(비과세 200만) → 실제 9.9%×800만=79.2만. 위탁가정=배당15.4%+양도15.4%.
  절세액 = 위탁가정 − 79.2만 ±1원.
- 경계: ISA 비과세한도 이내(실제 0, 절세=위탁가정 전액)·손실(절세 0).

### L-SAVE2 — 연금 절세액 (러프)
- 연금 수령세(나이 기반) vs 위탁가정. 절세액 부호·크기 손계산.

### L-SAVE3 — 위탁 불변식
- 위탁 계좌 절세액 == 0 (항상).

### L-SAVE4 — 합산 = Σ계좌
- combined 절세액 = Σ account 절세액.

각 단계 P1~P3는 해당 L-SAVE 통과 후 다음 탭.

---

## 알려진 한계 (UI 명시)
- 절세액은 **근사치**(ISA net 재투자·종합과세 가산 생략).
- 연금 1,500만 수령한도 초과세는 **은퇴 탭(P3)에서만** 반영.
- 배당금계산기(P4)는 후속.

## 미해결 / 결정 대기
- 종합과세 가산을 위탁 가정에 포함할지(정확↑·복잡↑) vs 생략(러프). → **생략으로 시작**, 필요시 추가.
- 연금 수령세 러프 산식 정확도 — `TaxEngine.after_tax_withdrawal` 연금 경로 재사용 가능.
