## 1. 데이터 소스 — SEC 13F (퍼블릭 도메인)

- 미국 운용자산 $100M↑ 기관은 분기마다 **Form 13F-HR**를 SEC EDGAR에 제출. 미국 정부 공문서 = **저작권 없음, 자유 이용**.
- 접근 = **SEC EDGAR**, **API 키 불필요**. 단 요청 헤더 `User-Agent: <앱이름 이메일>` 필수(SEC 규정), rate limit 10 req/s.
  - 제출 목록: `https://data.sec.gov/submissions/CIK##########.json` (CIK 10자리 0패딩)
  - 13F 정보테이블 XML: 해당 filing 폴더의 `*.xml` (보유: CUSIP·발행사명·금액(천$)·보유주식수·콜/풋 구분)
- **한계(필수 표기, UI 상단 고정 배너):**
  - **분기 1회 · 공시 마감 최대 45일 지연** → "현재 포폴" 아님, **"YYYY Qn 공시 기준"** 명시
  - **미국 상장 롱(주식)만** — 공매도·옵션 일부·채권·현금·비미국 자산 **안 보임**
  - 13F는 "스냅샷". 실제 매매 시점/평단 모름.
  - 면책: *"공개 13F 자료 가공, 정보 제공 목적, 투자자문·매매권유 아님."*

## 2. 파이프라인 — CUSIP→티커→symbol_master

13F는 티커가 아니라 **CUSIP**을 줌. 우리 엔진은 티커 기준 → 매핑 필요.

1. CIK별 최신 13F-HR fetch → 정보테이블 XML 파싱 → (CUSIP, 발행사명, 금액, 주식수) 리스트
2. **CUSIP→티커 매핑**: **OpenFIGI API**(무료 키, `POST /v3/mapping`, ID_CUSIP→ticker, 헤더 `X-OPENFIGI-APIKEY`). 배치 100건/req.
   - ✅ 키 = 로컬 `.env`의 `OPENFIGI_API_KEY`(gitignored, 추적X). 작동검증 완료(037833100→AAPL, 2026-06-25).
   - ⚠️ **시크릿 배포**: 값은 repo에 절대 커밋 금지. prod 워커 env + GitHub Action(`guru-resync-13f.yml`) repo secret `OPENFIGI_API_KEY`를 **오너가 별도 등록** 필요(값 아닌 이름만 문서화). 키없음=graceful no-op.
   - 대안(키 없이): SEC `company_tickers.json` + CUSIP 사전(불완전) → 매핑 실패분은 "미매핑"으로 표시하고 비중 계산서 제외.
3. 매핑된 티커를 **`symbol_master`**(data/meta/symbol_master.db)와 대조 → 우리 가격엔진 커버리지 확인. 미커버 종목은 별도 표기.
4. 비중 = 13F 보고금액 / 총합(매핑 성공분). 상위 N(기본 15) + 기타.

**저장 — 새 SQLite seed DB**: `data/meta/guru_holdings.db` (git 추적 seed).
- 테이블: `gurus`(cik, name, fund, slug, monogram, **stance** 1~10 낙관→비관 정렬순, stance_label), `filings`(cik, period, filed_at, accession), `holdings`(accession, cusip, ticker, name, shares, value, weight)
  - 목록 기본정렬 = `stance ASC` (낙관 왼쪽 → 비관 오른쪽). UI에 스펙트럼 범례 + 면책.
- ⚠️ **prod beat는 deploy 시 reset-hard로 되돌아감**([[reference-prod-deploy-access]], [[reference-symbol-master-seed-db]]) → 런타임 갱신 영속 안 됨.
  **갱신 = GitHub Action(분기)에서 빌드→커밋→배포**로만. `resync-symbols.yml` 패턴 재사용.
  - 워크플로 `guru-resync-13f.yml`: 분기(1·4·7·10월 중순 cron) EDGAR 폴 → 새 filing 있으면 DB 재빌드 → commit.

## 3. 백테스트 연결 (최근 성과)

- 대가 상세 페이지에 **"이 포트폴리오 백테스트"** → 상위 N 보유 비중을 고정비중 포폴로 구성 → 기존 `backtest_logic` 호출.
- 기간 선택(1·3·5Y). 결과 = 누적수익·MDD·변동성(기존 결과 컴포넌트 재사용).
- **가정 명시**: "13F 상위 N 종목을 공시 비중대로 보유·주기 리밸런싱 가정. 실제 펀드 성과와 다름."
- 미커버/미매핑 종목 비중은 현금 또는 비례 재정규화(택1, 기본=재정규화) — UI에 주석.

## 4. 화면 (탭 내부 — 모바일·웹 결 맞춤)

라우트 `/gurus`. 기존 페이지 디자인 토큰·공용 컴포넌트 재사용([[feedback-design-archetype-not-reskin]]).
- **목록 뷰**: 대가 카드 그리드(웹 2~3열 / 모바일 1열). 카드 = 아바타 + 이름 + 펀드 + 대표 보유 미니 비중바 + "YYYY Qn".
- **상세 뷰**: 상단 한계 배너 → 보유 테이블(종목·비중·금액) + 비중 도넛 + "백테스트" 버튼 → 결과.
- 모바일 = 세로 스크롤(스와이프 개편 안 함, 오너 확정). 항목 많으면 보유 테이블만 가로 스크롤 허용.

## 5. 이미지/저작권 (오너 위험판단 — §0.3)

- **이름·보유내역 = 사실(fact). 저작권/퍼블리시티 무관. 자유.**
- **사진**: Wikimedia 등 CC 라이선스 사진은 **저작권만 해결**, **퍼블리시티권(초상권)은 별개**.
  한국 2022 개정 부정경쟁방지법 = 국내 널리 알려진 인물의 초상 무단 **상업적** 사용 = 부정경쟁행위(2024 시정명령까지). 상업 앱 기능 = 위험.
- **AI 생성 이미지**: 실존 인물로 **식별 가능**하면 사진이든 일러스트든 **퍼블리시티권 동일 적용**. AI가 권리를 세탁하지 않음. "우연한 닮음"도 상업적 맥락서 청구 사유.
- **유튜버 사용 ≠ 안전 근거**: 유튜브 비평/보도 = 편집적 이용(공정이용 여지 큼). 상업 제품 기능은 맥락 다름(여지 작음).
- **권고:**
  - MVP = **모노그램/이니셜 아바타(예 "WB") 또는 추상 실루엣 = 0리스크.** 먼저 출시.
  - 정 얼굴 원하면: CC-BY 사진 + 출처표기 + 정보제공/면책 프레이밍 = 위험 낮음이나 **비0**. 순수 합법 보장 아님 → 오너 비즈니스 리스크 결정.
  - 식별 가능한 AI 실사/일러스트 = 위험 동일, 권장 안 함.

- 위치: **`market-card`(시장 지수) 바로 위**. HTML 한 블록으로 양쪽 커버:
  - 웹 = feat-sec(6카드) 다음 → 시장지수 사이
  - 앱 = mobile-doors(계산·시뮬레이션 기둥) 다음 → 시장지수 사이

## 단계 (Phase)

- **P1 데이터**: EDGAR 13F fetcher + 파서 + OpenFIGI 매핑 + `guru_holdings.db` seed 빌더(스크립트). 골든 대가 3명으로 검증.
- **P2 페이지**: `/gurus` 목록+상세(한계 배너·보유 테이블·도넛). 모노그램 아바타.
- **P3 백테 연결**: 상세 "백테스트" → backtest_logic, 결과 렌더.
- **P4 홈 카드**: 시장지수 위 진입 카드(웹·앱), Playwright 라이트/다크·모바일 검증.
- **P5 자동화**: `guru-resync-13f.yml` 분기 워크플로(빌드→커밋) + 새 공시 인앱 알림.
- **(보류) 이미지 고도화**: 오너 위험판단 후 사진/일러스트 여부.

## 검증

- 프론트: 로컬+Playwright `/gurus` + 홈 카드 라이트/다크·모바일, 콘솔에러 0([[feedback-design-archetype-not-reskin]] 검증규칙).

## 리스크/주의
