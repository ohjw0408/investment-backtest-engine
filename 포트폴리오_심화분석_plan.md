> **상태: v3(2026-06-26). 오너 2차 검수 반영. 설계 결정 8개 전부 확정 → 착수 가능.**

## 0. 목표 / 비전

현재 "포트폴리오 분석"은 **단일 경로(point estimate)** 만 보여준다 — 한 번의 시작~끝 백테스트(CAGR·MDD·연간수익률 1개 시계열). "운 좋은/나쁜 시점"에 좌우되고 장기투자의 통계적 안정성을 못 느끼게 한다.

**두 축으로 끌어올린다:**
1. **분석탭 심화** — 단일 경로 → **분포/롤링 기반**. "언제 시작했든" 관점. 핵심 메시지: *"10년 이상 투자하면 손실 확률이 거의 0이구나"* 를 사용자가 **눈으로** 느끼게.
2. **비교탭 동급화** — 분석탭의 깊이(연간수익률·배당·배당성장·MDD·롤링·드래그줌)를 비교탭에 이식, N개 포폴을 **같은 깊이로** 비교.

> 설계 원칙: ① 결과 불변(기존 백테/지표 회귀 0) ② 1 vCPU → 벡터화 ③ 합성 백필 구간 **항상 명시**(syn 플래그 재사용) ④ 분석·비교 **같은 엔진** 공유 ⑤ **모든 지표·그래프에 쉬운 설명 동봉**(아래 §2A — 오너 최우선 강조).

## 1. 현재 상태 (코드 실사, 2026-06-26)

### 분석탭 (`/backtest`, `backtest_logic.py`, `templates/backtest.html`)
- 비동기(celery). 반환: `metrics{cagr,mdd,sharpe,total_return,total_dividend,years,end_value}`, `history`(일별 PV), `annual_returns`(연), `annual_dividends`(연), 멀티계좌·세금·수수료.
- 차트 **드래그 확대 있음** (`chartjs-plugin-zoom`, backtest.html:1693·1948).
- **없음**: 롤링, 수익률 분포, underwater(MDD 시계열), 배당성장률, 손실확률, 일부 심화지표.

### 비교탭 (`/risk-return`, `risk_return_logic.py`, `templates/risk_return.html`)
- 동기 API `/api/portfolio/compare` → `compute_comparison`: **11지표**(cagr·vol·sharpe·mdd·sortino·div_yield·연최고·연최저·승률·베타) 표 + scatter + radar.
- **추세 겹쳐보기**(rrOvChart) — 전체기간·합성 점선·중앙값 규격화 완료. 데이터=`_portfolio_index_series`(일별 정규화 지수+`syn`).
- **없음**: 드래그 확대(오버레이), 연간수익률/배당 시계열, 배당성장률, 롤링, underwater, 상관행렬.

### 공통 자산(재사용)
- `chartjs-plugin-zoom` 전역 로드됨.
- `_portfolio_index_series(tickers, years=None)` = 일별 총수익 정규화 지수 + 합성 플래그(전체기간) → **롤링·분포의 데이터 소스**.
- `backtest_logic.history`(일별 PV) = 분석 롤링 소스.
- 합성 플래그(`volume=0`) = 장기 신뢰도 표기 재사용.

### ✅ 롤링 엔진 — 조사 결과 (오너 질문 답)
- **기존 엔진 존재**: `dividend_simulator._run_rolling`/`_iter_rolling_windows`(step_months, 실데이터→백필 2단), `portfolio_engine.prepare_for_rolling`(가격 1회 preload→윈도우는 slice), `attribution.analyze_rolling`, `multi_account_common`(부채꼴). 계산기 기본 step=**3개월**.
- **그러나 이식 불필요**: 우리 목표(기간별 손실확률·분포)는 **`_portfolio_index_series`의 총수익 인덱스에서 pandas 벡터화 롤링**으로 직접 산출 = 거의 무비용. 계산기 엔진은 윈도우마다 **풀 재시뮬**(적립·세금·리밸밴드 부채꼴 정밀도용)이라 무겁고 우리 용도엔 과함.
  - → **결정: 인덱스 기반 경량 롤링 채택.** 계산기 부채꼴 엔진 import 안 함. (적립·세금까지 반영한 롤링이 나중에 필요하면 그때 도입.)
- **시간/스텝**: 인덱스 기반이면 40년×6 horizon도 ms 단위 → 스텝 1/3/6개월 비용차 무의미. → **1개월(최세밀) 기본.** 비교탭(동기 API)도 인덱스 후처리라 타임아웃 위험 낮음.

## 2. 확정된 설계 결정 (오너 검수)

1. **합성 데이터 처리 = 옵션 A 확정.** 합성 포함 계산하되 합성 의존 구간/윈도우는 명시(점선·회색·"추정" 배지 + 툴팁에 합성%). syn 플래그 일관 사용.
2. **롤링 스텝 = 1개월.** 인덱스 기반이라 비용 무시 가능(스텝 무관). horizon = 1·3·5·10·15·20년.
3. **계산기 엔진 이식 안 함.** 인덱스 벡터화 롤링으로 충분(§1 답).
4. **몬테카를로 전망 = 기각**(투머치). 지표·그래프·롤링으로 충분.
   - **롤링 시각화 = 오차막대(box-whisker) 형식 확정.** 각 horizon의 결과 분포를 **막대=p25~p75, 수염선=p0~p100(최저~최고), 가운데 표식=중앙값(median)** 으로. 여러 포폴을 horizon별로 나란히 한눈 비교.
   - **퍼센타일 4단계 = 기본 0·25·75·100, 사용자가 각 단계 설정 가능**(예: 5·25·75·95로 변경). UI에 4개 퍼센타일 입력/프리셋.
5. **UI 패턴 확정**:
   - **분석탭** = 롤링만 추가 → **아코디언 없이 기존 결과 아래에 결 맞춰 쭉 추가**(롤링 차트·기간별 요약표·배당성장).
   - **비교탭** = 핵심지표 표 바로 아래 **아코디언 5개**: 수익률 · 변동성 · MDD · 배당 · 배당성장률. 각 아코디언 클릭 → **(좌) 해당 지표 연도별 꺾은선 겹침 그래프** + **(우) 롤링 결과 오차막대 모음** + **설명**.

6. **롤링 데이터원 = 총수익(배당 재투자) 인덱스.** ⚠️ 현 `_portfolio_index_series`는 `close`만 써 **가격수익(배당 제외)** — 그대로 쓰면 SCHD 등 배당주 롤링이 실제보다 낮게 나옴. P0에서 **배당 재투자 총수익 인덱스**(price_daily의 dividend/corporate_actions 반영)로 빌드. (추세 겹쳐보기 오버레이도 이 총수익 인덱스로 교체 → 정합.)
7. **롤링 기준 = 전체기간·거치식·배당재투자(개별 백테 설정과 별개).** 롤링은 "언제 시작했든"의 통계 관점이라 사용자의 적립금·리밸모드·세금·선택 구간과 무관하게 **전체 가용 기간**에서 산출. → 헤드라인 백테 CAGR과 롤링 중앙값이 정확히 일치하진 않음. **이 차이를 설명 카피로 명시**("롤링은 거치식·배당재투자·전체기간 기준의 분포").
8. **표본 부족 가드.** horizon이 가용(특히 실데이터) 기간보다 길어 윈도우가 적으면(예: 20년 롤링인데 실데이터 8년) → "표본 부족/합성 의존" 회색 처리 + 툴팁(n·합성%). 무의미한 막대 방지.

9. **비교탭 UX = 유입 경로로 2모드 분기**(2026-06-26 오너 논의). 같은 페이지·같은 컴포넌트, **초기 동작만** 다르게:
   - **예시 유입 모드**(preload 있음 = `mm_rr_preload`/`_rrExPreload`): 의도="고른 포폴 다 보여줘". 추세 겹쳐보기뿐 아니라 **정밀비교·전 그래프 자동 발화**(자동 `rrCompare`), 결과 영역으로 자동 스크롤, 입력 빌더는 접거나 하단으로(수정용 부차). → 하나의 **비교 대시보드**처럼 위→아래 완성된 상태로 도착.
   - **직접 입력 모드**(preload 없음): 의도="내 포폴 짜서 본다". 입력칸이 주인공, **자동발화 금지**(데이터 없음), 버튼 눌러야 결과. 입력칸 위·결과 아래.
   - **버튼 가시성**: 직접입력 모드에서 종목 많으면 "📊 정밀 비교하기" 버튼이 빌더 밑으로 밀려 화면 밖 → **sticky 버튼**(화면 하단 고정)으로 해결. (대안: 빌더 `max-height+내부스크롤`.)
   - **예시 유입 후 사용자가 종목 수정 시**: 매수정 자동 재발화는 무거움(11지표) → "변경됨 · 다시 비교" 배지/버튼(sticky)로 재실행 유도. 자동 재발화 안 함.
   - 적용 시점 = **P3에서 비교탭 UI 갈아엎을 때 같이**(별도 작업 안 함).

### 2A. 설명(카피) 원칙 — **오너 최우선** 🔴
모든 지표·차트에 **쉬운 설명 필수**. 각 항목마다:
- **뜻**: 이게 뭔지 한 줄(전문용어 풀어서).
- **방향**: 높은 게 좋은가/낮은 게 좋은가 (또는 "취향").
- **롤링 퍼짐(분산)의 의미**: 퍼진 게 좋은가/뭉친 게 좋은가 — **지표마다 다름**:
  - 수익률: 상방 뚫린(위로 퍼진) 게 매력적이지만, **어떤 사람은 예측 가능하게 좁게 뭉친 걸 선호** → "정답 없음, 성향" 명시.
  - 변동성·MDD: 낮고 뭉친 게 거의 항상 선호.
- **어려운 개념은 특히 쉽게**: 상관행렬·Sortino·capture 등은 비유/한 줄 예시로. (예: 상관 = "같이 움직이나, 따로 노나" → 1에 가까우면 분산효과 약함.)
- 합성 구간 주의 문구.

## 3. 단계 (Phase)

> 1순위 = **P1(비교탭 드래그 확대)** — P0와 독립이라 단독 선출시 가능(빠른 가치). 이후 P0→P2→P3→P4.

### P0. 공용 롤링 엔진 토대 ✅ 완료(2026-06-26)
- ✅ **총수익(배당 재투자) 인덱스** — 결정#6. `_ticker_series`(app.py)가 close→**배당 재투자 TR**로(`tr*=(close+dv)/prev`). ⚠️ price_daily는 이미 분할조정이라 split 곱 금지(SCHD 2024 3:1 가짜점프 버그 발견·제거). 검증: SCHD price 3.85→TR 6.15x·QQQ 12.6→14.27x(저배당 근접). 오버레이도 자동 TR.
- ✅ **`modules/rolling.py` 신규**(순수함수, 월말 리샘플 기준):
  - `rolling_cagr(points, years)` → 직전 N년 CAGR 월별 시계열.
  - `horizon_table(points, horizons, percentiles)` → horizon별 {n, **loss_prob**, median, syn_frac, pcts{0/25/75/100…}} ← 오차막대·요약표 소스.
  - `drawdown(points)` → underwater + 최대낙폭·저점·**회복기간**·최장침수.
  - `real_adjust(points, infl=0.02)` → 인플레 실질 인덱스(syn 보존).
- ✅ **골든 테스트** `tests/test_rolling.py` 8개 PASS(상승=손실0·하락=손실1·표본부족·낙폭회복·실질할인·syn).
- 잔여(P2서): 분석탭 `history` 어댑터(분석은 단일경로라 rolling 입력은 인덱스 재사용 가능).

### P1. 비교탭 추세 겹쳐보기 — **드래그 확대** ✅ 완료(2026-06-26, 미푸시는 오너 승인 시)
- rrOvChart에 `chartjs-plugin-zoom` drag 줌(mode x) + **↺ 초기화 버튼**(resetZoom + 기본 5년 복귀).
- **이벤트 프리셋 줌**: 닷컴(2000–02)·금융위기(2007–09)·코로나(2020)·'22 긴축 원클릭(가용범위 클램프, 데이터 없으면 토스트).
- 드래그 안내 힌트 추가.
- 검증(Playwright): zoom 플러그인 로드·프리셋4·코로나 범위 적용·드래그줌 x스케일 변화(0-252→69-154)·초기화 복귀·콘솔0.
- 변경=`risk_return.html`. ⏳ 모바일 핀치 별도(후속).

### P2. 롤링 + **분석탭 심화** (기존 결과 아래 쭉) ✅ 완료(2026-06-26)
- ✅ **① 기간별 결과 요약표(킬러)** — horizon(1/3/5/10/15/20y)별: 손실확률·중앙 수익률·최악·최고·표본수(`btHorizonTable`). 손실확률 0근접=초록·≥30%=빨강. 쉬운 설명 동봉.
- ✅ **② 롤링 분포 오차막대** — box-whisker(박스=p25~75 / 수염=p0~p100 / 중앙값 가로선). 커스텀 플러그인 `_btBoxPlugin`(시간축 어댑터 없음 → category). "기간 길수록 좁아지고 0 위로" 시각 확인.
- ✅ **③ 롤링 CAGR 시계열** — 1/3/5/10y 다선(범례 토글, 1년 기본 숨김). 날짜 합집합 category축.
- ✅ **④ underwater(낙폭) 차트 + 회복** — 음영 area + 최대낙폭·저점→회복·최장침수 통계.
- ✅ **⑤ 배당성장률** — 기존 `renderDividendGrowthChart`(클라, annual_dividends)가 이미 충족 → 백엔드 중복 미추가.
- ✅ **⑥ 인플레 실질 토글** — 명목/실질 세그(`bt-seg`) + 물가율 input(기본 2%·변경가능). 실질 = CAGR 디플레이트 (1+nom)/(1+infl)-1, 프런트 처리(백엔드는 infl만 전달).
- ✅ 합성 구간 명시(결정#1) — syn_frac>0.5 행·박스 회색 + ⓘ(윈도우 합성%), syn_overall>0.6 골드 경고 배너.
- 산출: 신규 `modules/tr_index.py`(Flask-free 공용 TR 인덱스 빌더, app `_portfolio_index_series` 코어 미러·NULL홀가드) + `backtest_logic.compute_rolling_analysis`(단일+멀티계좌 자본가중) + `backtest.html`(HTML 섹션 + JS 6렌더 함수).
- ⚠️ **데이터 기준 = 전체기간+합성 회색표기(오너 결정 2026-06-26, 결정#1 유지).** SPY/TLT 등은 합성백필이 1928~ → 헤드라인 MDD -85%(1932 합성)·전행 회색. 실데이터한정 대안은 오너 기각.
