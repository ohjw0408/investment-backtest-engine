## 1. 문제

가상데이터(use_synthetic)로 상장 짧은 ETF 장기 시뮬 시, 상장 전 구간을
**종목별 독립 GBM**으로 생성(`_load_with_per_window_synthetic`이 `seed=hash(code+window)`로
종목마다 따로 난수) → 합성구간 종목 간 **상관 ≈ 0**. 실측:

| | GLD-SCHD | GLD-SPY | SCHD-SPY |
|---|---|---|---|
| 합성구간(1975~2000) | -0.01 | -0.05 | **0.03** |
| 실데이터(2005~2024) | 0.02 | 0.09 | **0.89** |

실 SCHD-SPY=0.89인데 합성 0.03. **상관 기반 분산효과 시험이 핵심 목적**인 앱에서 치명적.
증상: 합성구간 리밸런싱 비현실적 이득(독립자산→변동성 수확 극대화, 40년 +25%·비단조),
MDD 리밸 무반응. (실 상관데이터 18년은 +3%로 정상.)

## 2. 목표

합성 pre-history를 **추정된 상관행렬에 부합하도록 다변량 생성**. 특히 합성종목이
**같은 날 실데이터가 있는 종목의 실제 등락을 따라가도록(조건부)** 하여 synth-real 상관(SCHD-SPY=0.89)까지 재현.
시장지수 같은 외부 팩터 불필요 — 포트 내 종목들끼리의 상관행렬만 사용.

## 3. 모델 (조건부 다변량 정규/-t)

### 3.1 통계 추정 (1회, 포트 단위)
- **공통 데이터 구간** = 전 종목이 데이터(실+백필) 가진 날의 교집합(보통 max(actual_start) 이후).
- 그 구간 **일수익률**로 평균벡터 `μ`와 공분산 `Σ`(→ 상관) 추정. (월 아닌 일 단위로 충분한 표본 확보.)
- 표본 부족(<~252일)·특이행렬 → 대각 수축(shrinkage)·ridge·경고로 폴백.

### 3.2 조건부 생성 (윈도우별, 시간 전진)
합성일 t마다 종목을 분할:
- `R(t)` = 그날 데이터(실/백필) 있는 종목 → 실제 수익 `a_t`(알려짐).
- `S(t)` = 그날 합성 필요 종목 → 조건부 샘플.

조건부 다변량 정규:
```
r_S(t) | r_R(t)=a_t  ~  N( μ_S + B·(a_t − μ_R),  Σ_cond )
   B       = Σ_SR · Σ_RR⁻¹
   Σ_cond  = Σ_SS − Σ_SR · Σ_RR⁻¹ · Σ_RS
```
- 구현: `r_S(t) = μ_S + B(a_t − μ_R) + L·z_t`,  `L = chol(Σ_cond)`,  `z_t` = iid 표준-t(df=5)/T_SCALE 벡터(두꺼운꼬리 유지, t-copula 근사).
- `R(t)` 공집합(전 종목 합성)인 날 → 무조건 결합: `r_S = μ_S + L0·z_t`, `L0=chol(Σ_SS)`.

→ **synth-synth 상관**(결합 draw) + **synth-real 상관**(조건부 평균이 실 등락 추종) 둘 다 재현.
예: 1971~2003 구간 R={SPY 실}, S={SCHD,GLD} → SCHD 합성이 SPY 실 등락을 corr 0.89로 추종.

### 3.3 구간(segment) 분할 = 효율
조건 집합 R/S는 actual_start 경계에서만 바뀜 → 종목 k개면 **≤ k개 구간**.
구간마다 `Σ_RR⁻¹`·`B`·`chol(Σ_cond)` 1회 precompute 후 그 구간 날들 **벡터화**.
연산 = 구간당 O(k³)(역행렬·Cholesky), k≤5라 무시 가능. (오너 우려한 폭증 없음 — k 작음.)

### 3.4 가격 재구성 + FX anchor
각 종목 합성수익 `r_i(t)`로 가격 재구성. anchor = `actual_start`의 **FX 실가격**(BUG-SYNTH-FX 수정 유지),
끝값을 anchor에 연결해 역방향 적분(`price(t-1)=price(t)/(1+r_i(t))`). 결정론 seed 유지.

### 3.5 드리프트(평균) 처리
`μ_S`(합성종목 평균)는 공통구간(최근) 표본이라 높을 수 있음. 조건부 평균이 실종목에 묶여
대부분 완화되나, backstop으로 `MAX_SYNTH_MU_MONTHLY` 캡을 `μ_S`에 유지(과대 외삽 방지).
→ **결정 사항**: μ_S 그대로(조건부에 의존) vs 캡 vs 시장수축. 추천=캡 backstop 유지.

## 5. 폴백·엣지

- 공통 데이터 구간 <~252일 → Σ 대각수축 또는 독립 폴백 + 경고.
- `Σ_RR` 특이(공선성) → ridge/pseudo-inverse.
- 어떤 합성일에 실종목 0개(전부 합성) → 무조건 결합 draw.
- 종목이 공통구간서 데이터 전무 → 그 종목만 독립 폴백 + 경고.
- 백필 데이터도 "데이터 있음"으로 간주(오너 결정: 실+백필).

## 6. 검증 (구현 후 필수)

1. **상관행렬 복원(결정 지표):** 재생성 후 합성구간 empirical corr ≈ 추정 Σ corr(허용오차).
   SCHD-SPY 0.03 → **~0.89 근처**, GLD-SPY 낮게 유지.
2. **리밸 정상화:** 합성 GLD+SCHD+SPY 40년 none/band1/band30 스프레드가 실 상관데이터 패턴
   (소폭·단조)과 유사. 현재 +25%·비단조 → 목표 +몇%.
3. **수익 비폭발:** BUG-SYNTH-FX 회귀 불변(GLD/SCHD 배수 현실, 포트 시장급).
4. **비합성 불변:** QQQ+SPY 실데이터 결과 동일.
5. **anchor 연속성:** 경계 점프<5 (`test_synthetic_anchor_fx` 그린).
6. 전체 회귀 그린 + 신규 테스트(상관행렬 복원·리밸 정상화·조건부 생성기 단위).

## 7. 리스크·트레이드오프

- 조건부 다변량은 **선형(피어슨) 상관**만 재현(비선형·꼬리의존 근사). 0보다 압도적 개선.
- 공통구간이 짧으면 Σ 추정 불안정 → 수축 폴백 필요(표본 적은 신생 ETF 조합 주의).
- 합성구간 실종목 성분은 단일 실역사 경로(랜덤 아님) — 윈도우마다 다른 구간이라 변동 확보,
  backtest로는 오히려 현실적(진짜 위기 포함).
- `MAX_SYNTH_MU` 캡·anchor-FX 수정 유지.
- k 큰 경우 O(k³)이나 k≤5 가정 하 무시 가능. (k 매우 클 때 경고/제한 고려.)

## 9. 미해결 결정 사항 (오너 확인)

### 9.1 μ_S(합성종목 평균수익) 처리
- **무엇:** 조건부 평균 `μ_S + B(a_t−μ_R)`의 `μ_S`는 합성종목의 표본 평균. 이게 짧은 최근
  불장 구간(예 SCHD 2011~)에서 추정되면 과대 → 합성 과거가 비현실적으로 우상향할 수 있음.
- **선택지:**
  - (a) **캡 backstop 유지** — `μ_S`를 `MAX_SYNTH_MU_MONTHLY(≈연8%)`로 상한. 조건부가
    대부분 묶어주고, 극단만 캡이 막음. **이미 배포된 안전망 재사용.**
  - (b) 조건부에만 의존 — μ_S 그대로. 실종목(R)에 강히 묶이면(예 SCHD가 SPY에 0.89) μ_S 영향
    작아 무방하나, R이 약상관이거나 전부 합성인 날엔 μ_S 과대가 그대로 샘.
  - (c) 0/시장수축 — μ_S를 0이나 시장평균으로 강제. 보수적이나 자산 고유 드리프트 소실.
- **트레이드오프:** (a)는 안전·단순하나 캡이 인위적. (b)는 순수하나 약상관 구간서 폭발 위험.
  (c)는 과도 보수.
- **추천: (a) 캡 backstop 유지.** 조건부가 평소엔 알아서 잡고, 캡은 드문 극단 방어. 위험 대비 단순.

### 9.2 공통 데이터 구간(상관행렬 추정 표본) 정의
- **무엇:** Σ를 어느 구간 일수익률로 추정하나. 전 종목이 데이터를 가진 날의 교집합이 기본.
- **선택지:**
  - (a) **엄격 교집합** — 모든 종목이 데이터 있는 날만(보통 가장 늦게 상장한 종목 이후).
    상관 추정이 일관되나, 신생 종목 하나 있으면 표본이 짧아짐(예 한 종목 2020 상장 → 5년치만).
  - (b) 쌍별(pairwise) 추정 — 각 종목쌍마다 그 둘이 겹치는 최대구간으로 상관 산출 후 행렬 조립.
    표본 더 쓰나 행렬이 **양의정부호(PSD) 깨질 수 있음** → 보정(nearest-PSD) 필요.
  - (c) 짧은 종목만 수축 — 교집합 기본, 표본 부족 종목은 상관을 대각(독립)으로 수축.
- **트레이드오프:** (a) 단순·안전하나 표본 손실. (b) 정보 최대지만 PSD 보정 복잡·구현 위험.
  (c) 절충.
- **추천: (a) 엄격 교집합 + 표본<~252일이면 수축 폴백.** 단순·안정. (b)는 추후 고도화.

### 9.3 두꺼운꼬리(fat tail) 분포
- **무엇:** 조건부 잔차 `z_t`를 어떤 분포로 뽑나. 실수익은 정규보다 꼬리가 두꺼움(폭락/폭등 잦음).
- **선택지:**
  - (a) **다변량-t 근사** — `z_t`를 표준 t(df=5)/T_SCALE로 뽑아 `L·z_t`. 기존 합성과 동일 df,
    폭락 사건 보존. 상관구조는 L이 유지(t-copula 근사, 엄밀한 다변량-t 아니나 충분).
  - (b) 정규 — `z_t` 표준정규. 단순·수학 깔끔하나 꼬리 얇아 위기·MDD 과소.
- **트레이드오프:** (a)는 현실적(MDD·꼬리위험 보존)이나 t-copula가 엄밀한 결합분포는 아님(근사).
  (b)는 분산효과 검증엔 무방하나 극단위험 과소평가.
- **추천: (a) 다변량-t 근사.** MDD·위기 시나리오가 분산효과 시험의 핵심이라 꼬리 보존 필요.

### 9.4 DB경로(`generate_and_save`) 수정 시점
- **무엇:** 합성엔 두 경로 — 윈도우 인메모리(`_load_with_per_window_synthetic`, **계산기·이 버그**)와
  DB 테이블(`generate_and_save`, `get_price(allow_synthetic)` 폴백·일부 retirement).
- **선택지:**
  - (a) **후순위** — 주버그(윈도우 경로)만 먼저. DB경로는 단일종목 독립 생성이라 조건부 다변량을
    못 담음(구조 다름) → 별도 설계 필요, 지금은 캡·경고만.
  - (b) 동시 — DB경로도 조건부로. 단일종목 API라 결합 생성과 안 맞아 대규모 재설계.
- **트레이드오프:** (a) 핵심 버그 빨리 해결·범위 좁음. DB경로는 계산기서 안 쓰여 영향 작음.
  (b) 완전하나 범위 폭증·위험.
- **추천: (a) 후순위.** 계산기/백테/은퇴 주경로는 윈도우 경로라 이걸로 충분. DB경로는 별도 과제.
